## So You Need Some Help?
Sometimes we get stuck with an error or a problem when coding and need some help from outside sources. One way to go about doing so, is to go to websites such as Stack Overflow and post your question there. Stack Overflow is a forum that allows you to ask questions related to coding in which any user can reply to your post attempting to answer your questions. Now before you start posting every single minute problem you run accross on one website, you should first take some time to yourself to exhaust other options. By that, I mean it's highly likely someone else came accross a similar problem years ago and someone already posted a solution to that problem. That's why its always good to search on google or the documentation of your programing language to see if your question has already been asked. In the end, you'll be doing yourself and others a favor by not wasting time, cause as we all know, time is precious.

## The Downfalls of a Bad Question
Speaking of the website Stack Overflow, there's a system in place where users can upvote or downvote your questions. It can be seen as a way to gauge what others think about your question using a numerical score. For example, if you've posted a question and it recieved a ton of downvotes, it could be an indication that your question is lacking in some way. 

Let's take a look at [this](https://stackoverflow.com/questions/22209729/what-does-do-mean-in-python) post titled "What does != do/mean in python". The user posted the following snipet of code and asked why it prints true:
```python
print 2 and 1-2 !=3
```
This post has a score of -17 which means it's been downvoted by multiple people, but what was the problem? It's just far too simple. You can answer it yourself by looking at python documentation about comparisons, and a user even provided a link to it in the comments. This is a problem that doesn't warrant its own question.

## What You Can Do to Ask a Better Question
Now what makes a question good? A good question is one that hasn't been posted multiple times before thats both clear and consise. It should explain what your issue is on what platform and for which language. One such example is this post [here](https://stackoverflow.com/questions/503093/how-do-i-redirect-to-another-webpage). The user asks "How can I redirect the user from one page to another using jQuery or pure JavaScript?". This question was posted 11 years 11 months ago, which at the time was probably one of the only questions of its type. That means its not a needless repost and was an actual problem people were encountering. It has a total of 7017 upvotes which shows how important it was at the time of posting. So what can we take from this? Make sure your question hasn't been asked before and that you've tried answering the question yourself first. If you do have to post a question, be clear, consise, and describe the problem you're encountering as well as you can. Include information thats esssential for others to know. Hope this helps and good luck in your journey!
