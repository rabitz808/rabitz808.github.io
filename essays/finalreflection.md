## Taking ICS 314
The first half of this class got me familiar with javascript and the second half of this class got me familiar with semantic ui, react, and meteor. As someone with little experience in these topics, the class itself was really well structured. We started off with the basics before moving on to the more harder topics. The best part was that what we learned in class all complemented each other when it came to website development. While it's great to learn all these new ideas for web development, these ideas can also be applied as general knowledge outside of web development.

## Development Environments
Outside of web development, the environment you use will be an essential stepping stone in reaching new heights. What I mean by this is that you can use the same development environment for different programming languages. I really enjoyed using IntelliJ as the program for compiling source code as it has an easy to use interface and once you have it all set up, you don't need to tamper with it after that. I'm planning on continuing to use IntelliJ when I take other classes even if they don't use a language such as javascript.

## Coding Standards
The next best thing aside from the development environment that I found helpful was learning about coding standards. With IntelliJ it's simple to install eslint to monitor and correct the format of your code. This makes it easy to read and catch any errors that you come across when compiling code. It doesn't just pertain to javascript, but can also be applied to all types of languages. You can use it for stuff like python or even lua. Following a coding standard will keep my work consistent when developing on a larger code database and is something I'll always remember to follow.
